Towaria

30 September―25 October, 2024

Introduction

In Towaria, your goal is to get as far as you can, survive the waves of monsters which become more and more difficult with each level. Each level has a unique layout so your strategies may adapt based on the level you're playing on.

This was a school project to learn about mobile development.

The systems others interface with to control different internal processes. This also involved intergrating a lot of code.

My role within the project

I mainly focussed on programming the "behind the scenes" logic of the game, so to speak. Think of the random map generation, or how the save data is structure, saved and loaded.

Summarised, I made the code which interfaces a lot with other people's code, making their lives easier. This involved some code intergration on my part as well.

Challenges faced

One of the main challenges was figuring out how to communicate with other people, whilst working on the same project. And still have a cohesive codebase. The final code isn't what I'd love it to be, but the product itself I am very happy with.

A challenge which we thought to run into more than we did, was the limited performance of android. But it seems like all the code we wrote was already quite optimised. Though, this didn't stop us from optimising things further. The main bottleneck is rendering, though that is beyond my knowledge at this time.
